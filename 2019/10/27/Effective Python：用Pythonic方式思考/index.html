Effective Python：用Pythonic方式来思考

发表于 2019-10-27 | 更新于: 2019-10-27 | 分类于 python

本系列为《Effective Python:编写高质量Python代码的59个有效方法》的读书笔记。

Python开发者用Pythonic这个词来描述具有特定风格的代码。这种风格是大家在使用Python语言进行编程并相互协作的过程中逐渐形成的习惯。

1. 确认自已所用的Python版本

Python通常使用的有Python2 和 Python3两个版本，其中Python2的功能开发已经冻结，即将在2020年1月1日正式“退休”，因此建议使用Python3来开发下一个Python项目。

命令行

Python 3.7.4

程序内部

sys.version_info(major=3, minor=7, micro=4, releaselevel='final', serial=0)
3.7.4 (tags/v3.7.4:e09359112e, Jul  8 2019, 20:34:20) [MSC v.1916 64 bit (AMD64)]

2. 遵循PEP 8风格指南

PEP8 即 Python Enhancement Proposal #8 (8号Python增强提案)，是针对Python代码格式而编订的风格指南，用以指导如何撰写清晰的Python代码。PEP8会随着Python语言持续更新，详细见https://www.python.org/dev/peps/pep-0008/

命名

  • 函数、变量即属性用小写字母拼写，各单词间使用下划线相连，例如lowercase_underscore
  • 受保护的实例属性，应以单个下划线开头，例如， _leading_underscore
  • 私有的实例属性，应该以两个下划线开头，例如，__double_leading_underscore
  • 类与异常，应该以每个单词首字母均大写的形式来命名， 例如，CapitalizedWord
  • 模块级别的常量，应全部采用大写字母来拼写，例如，ALL_CAPS
  • 类中的实例方法（instance method），首个参数名应命名为self，以表示该类自身
  • 类方法（class method）首个参数应命名为cls

表达式和语句

  • 采用内联形式的否定词，例如，应为if a is not b 而不是 if not a is b
  • 检查some为[]或’’等空值，应使用if some 或if not some，而不是if len(some) == 0。Python会将空值评估为False，非空值评估为True
  • if，for，while及except等复合语句应分成多行来书写
  • import语句总是放在文件开头，以标准库模块、第三方模块、自用模块进行排序
  • 引入模块，应总是使用绝对路径，例如 from bar import boo，而不是直接import boo
  • 如果一定要以相对名称使用import语句，明确标注：from . import foo

3. 区别bytes、str与unicode

如果对unicode等字符代码或utf-8,utf-16等字符编码不清楚的，可以参考下述内容：

  • Unicode和utf8的关系是什么？
  • Unicode与UTF-8的区别
  • 计算机内存和文件中的UNICODE字符

Python3 和 Python2 的区别

  • Python3 中：byte 表示包含原始字节的8位值，str 表示包含unicode字符
  • Python2 中：str 表示包含原始字节的8位值， unicode 表示包含unicode字符

encode 和 decode

Python3的str实例和Python2的unicode实例都没有和特定的二进制编码形式相关联。要将Unicode字符保存为二进制数据，须使用encode方法；将二进制数据转换成unicode字符，则必须使用decode方法。

良好实践

  • 编写Python程时，要把编码和解码操作放在逻辑最外层来做。程序的核心部分使用Unicode字符类型（即Python3中的str和Python2中的unicode），且不要对字符编码做任何假设。

  • 在开发中，通常需要处理原始字节的8位值或者Unicode字符，需要编写两个辅助函数（helper）函数来实现两种情况的转换。

    Python2中也可以编写同样功能函数，只不过对应类型变换位str和unicode

注意

  • 在Python2中，如果str只包含7位ASCII字符，则unicode和str似乎成为同一类型：

    • 可以用 + 将 str 和 unicode 进行连接
    • 可用 == 和 != 在str实例和unicode实列之间比较
    • 在格式字符串中，可以用’%s’等形式来代表unicode实列

    但在Python3中，bytes 和 str 绝不等价，即使是空字符串，所以在传入字符序列是必须注意类型

  • 在Python3中，通过内置open函数获取文件句柄，该句柄会默认采用 utf-8 编码格式来操作文件，而在Python2中文件操作则默认采用二进制形式。

    下面方法在Python2中可以正常运行，但在Python3中会报错。

    ---------------------------------------------------------------------------

TypeError                                 Traceback (most recent call last)

<ipython-input-3-090c1279f984> in <module>
    2 
    3 with open('random.bin', 'w') as f:
----> 4     f.write(os.urandom(10))

TypeError: write() argument must be str, not bytes

    这是因为Python3在open函数中添加了encoding参数，改参数默认值为’utf-8’。因此在文件句柄上的read和write操作则必须传入为unicode字符的str实例，而不能是bytes实例。
    为解决这个问题，可使用二进制写入模式（’wb’）来开启待操作文件。

    从文件中读取也有这种问题，是用’rb’模式打开文件即可。
