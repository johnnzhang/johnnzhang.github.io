Python基础--模块基础

发表于 2019-10-31 | 更新于: 2019-10-31 | 分类于 Python基础

每一个以.py为扩展名的Python源码文件就是一个模块。模块的导入就是载入一个.py文件并读取该文件中的内容。

1. 模块的导入和重载

模块的导入

模块的导入主要有以下步骤：

  • 找到对应模块文件
  • 将其编译成字节码
  • 运行代码

从上面的步骤中我们知道，模块的导入会运行模块文件中的代码，但是只会运行一次。

我们新建一个test.py,添加简单的打印内容:

test import module

第二次import则没有相关输出。

模块的重载

若想再次运行test模块，则需要对test模块进行重载。

test import module





<module 'test' from 'F:\\work\\jupyter\\learning\\Python学习手册\\test.py'>

可以看到reload函数载入test模块并重新运行，如果在另一个窗口中修改并保存了它，则会反应出修改的变化。
需要注意的是，reload只能重载已经载入的模块，若reload未载入的模块，则会出错。

reload是不可传递的，reload只会重载当前的module文件，而不会重载该module所导入的其他文件。

---------------------------------------------------------------------------

从导入relaod函数可以看到，还可以使用from进行模块的导入。与import不同的是，from导入的属性可以直接使用，import则是导入了模块的名字，需要使用module.attribute的格式进行调用。除此之外，两者并没有其他区别，都需要运行导入的module文件。

通过from module_name import * 可以导入module_name中的所有属性

模块的显要特性：属性

模块往往就是变量名的封装，被认作是命名空间；在一个模块中定义的变量名可以认为是该模块的属性。
导入模块即获得了模块中在顶层定义的所有变量名。
内置函数dir可以显示获得了模块中的哪些属性（变量）。

['__builtins__',
 '__cached__',
 '__doc__',
 '__file__',
 '__loader__',
 '__name__',
 '__package__',
 '__spec__',
 's']

从dir的返回的结果我们可以看到有属性’s’以及一些以双下划线开头和结尾的属性，这些属性是Python预定义的内置变量名。

2. 使用exec运行模块文件

上面提到import和from都会运行相应的module文件，python还提供了一种运行py文件中代码的方法，即exec。

exec对py文件的调用并不会导入模块，而是相当于把文件内容复制到了exec的地方，并进行调用。
这会产生一个问题是，由于exec对文件内容的直接调用，会对同名变量进行覆盖。

100
test import module
'test import module'

从上述可以看到，exec的执行对变量s进行了覆盖。
