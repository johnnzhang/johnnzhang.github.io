个性化配置

hexo 基本配置

hexo的配置文件为blog\_config.yml，其中主要配置有：

  • site
    • title：blog标题
    • subtitle：blog副标题
    • description：blog描述，用于SEO
    • author：blog作者姓名
    • language：blog使用的语言
    • timezone：blog时区
  • theme：blog使用的主题，我的blog使用的主题是next
  • deploy
    • repo：部署到的git仓库地址
    • branch：指定仓库的分支

Front-matter

Front-matter是blog文章对应的md文件的最上方以—分隔的区域，用于指定每篇blog的特殊设置。

  • layout：布局，有三种默认布局：post，page，draft
  • title：blog显示的文章题目
  • data：文章创建日期
  • updated：文章更新日期
  • comments：是否开启文章的评论功能
  • tags：标签
  • categories：分类，具有顺序性和层次性

page

如果想另起一页（页面？）test（访问路径为xxx.xxx/test），可以使用

系统会自动在blog\source文件夹下创建test文件夹，以及test文件夹中的index.md文件。

  1. 添加categories页

    1. 新建categories页，命令为

      hexo new page categries

    1. 在blog\source\categories\index.md的—分隔区域添加

      type: "categories"

    2. 编辑blog使用主题的_config.yml文件，将menu中的

      #categories: /categories/ || th 的注释去掉。

    同理，可按上述步骤添加tags、about页。

  2. 添加自定义页，则需要在主题的_config.yml文件中的menu添加对应的自定义的项进行匹配。

  3. 修改页的顺序，直接修改menu中项的顺序即可。

draft

draft用来提供草稿的功能（暂时不在blog中展示），

  1. 新建draft

    hexo new draft test_draft

    上述命令在blog/source/_draft中新建test_draft.md文件。

  2. 本地预览draft

    hexo server --draft

  3. 发布draft

    hexo public draft test_draft

多终端更新blog

  1. hexo机制
    hexo d 上传部署到github的是hexo编译后的文件，即同步的是根目录下的.deploy_git文件夹。
    如果需要多终端工作，则需要同步博客的相关源文件。
  2. 可以利用同步到github的仓库创建同步源文件的分支，例如创建sourece分支。 也可以将源文件的git仓库部署到自己的服务器上。

    注意：如果有clone过theme中的主体文件，例如我的blog中使用的next主题，需要将对应文件夹theme\next\.git删除，并同时执行git rm -r -cached theme\next\再执行相关git同步操作
